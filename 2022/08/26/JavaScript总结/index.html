####==DOM属性==

  • elem.hasAttribute(name) — 检查特性是否存在。
  • elem.getAttribute(name) — 获取这个特性值。
  • elem.setAttribute(name, value) — 设置这个特性值。
  • elem.removeAttribute(name) — 移除这个特性。
  • elem.attributes — 所有特性的集合。

==创建和插入==

  • 创建新节点的方法：

    • document.createElement(tag) — 用给定的标签创建一个元素节点，
    • elem.cloneNode(deep) — 克隆元素，如果 deep==true 则与其后代一起克隆。

  • 插入和移除节点的方法：

    • node.append(...nodes or strings) — 在 node 末尾插入，
    • node.prepend(...nodes or strings) — 在 node 开头插入，
    • node.before(...nodes or strings) — 在 node 之前插入，
    • node.after(...nodes or strings) — 在 node 之后插入，
    • node.replaceWith(...nodes or strings) — 替换 node。
    • node.remove() — 移除 node。

    文本字符串被“作为文本”插入。

  • 这里还有“旧式”的方法：

    • parent.appendChild(node)
    • parent.insertBefore(node, nextSibling)
    • parent.removeChild(node)
    • parent.replaceChild(newElem, node)

==样式和类==

  • 管理class
    • className — 字符串值，可以很好地管理整个类的集合。
    • classList — 具有 add/remove/toggle/contains 方法的对象，可以很好地支持单个类。
      • elem.classList.add/remove(class) — 添加/移除类。
      • elem.classList.toggle(class) — 如果类不存在就添加类，存在就移除它。
      • elem.classList.contains(class) — 检查给定类，返回 true/false。
  • 改变样式
    • style 属性是具有驼峰（camelCased）样式的对象, 它对应于 style 特性（attribute）中所写的内容。

==元素宽度、高度、几何特征==

  • offsetParent — 是最接近的 CSS 定位的祖先，或者是 td，th，table，body。
  • offsetLeft/offsetTop — 是相对于 offsetParent 的左上角边缘的坐标。
  • offsetWidth/offsetHeight — 元素的“外部” width/height，边框（border）尺寸计算在内。
  • clientLeft/clientTop — 从元素左上角外角到左上角内角的距离。对于从左到右显示内容的操作系统来说，它们始终是左侧/顶部 border 的宽度。而对于从右到左显示内容的操作系统来说，垂直滚动条在左边，所以 clientLeft 也包括滚动条的宽度。
  • clientWidth/clientHeight — 内容的 width/height，包括 padding，但不包括滚动条（scrollbar）。
  • scrollWidth/scrollHeight — 内容的 width/height，就像 clientWidth/clientHeight 一样，但还包括元素的滚动出的不可见的部分。
  • scrollLeft/scrollTop — 从元素的左上角开始，滚动出元素的上半部分的 width/height。

AJAX / webpack / babel

####==babel webpack 用法==

#####!!

  • 学会到 babel / webpack官网 查找 对应操作 / loader / plugins … 的使用文档
  • 多写

#####babel

  • babel 可以编译大部分 ES6 语法
  • ES6 Module 语法需要结合 webpack 处理
  • 官方文档 查 babel 使用方法
  • 一般在Cli(命令行) / webpack 使用

#####babel 使用流程

  • 安装node.js
  • 初始化项目 npm init 生成 package.json 文件
  • 安装babel 需要的包 @bable/core @babel/cli @babel/preset-env
  • 在 package.json 文件添加 执行编译的 命令 “build”: “babel src -d dist”
  • 创建配置文件 .babelrc 添加 “presets”: [“@babel/preset-env”]
  • 编译 npm run build

#####Webpack 使用流程

  1. npm init
  2. 安装webpack 需要的包 webpack / webpack-cli
  3. 在package.json 添加执行编译的 命令 “webpack” : “webpack –config webpack.config.js” / “webpack” : “webpack”（默认）
  4. 创建配置文件 （默认 webpack.config.js） 并配置
  5. 打包 并测试 npm run webpack (这里的webpack 对应 第三步 babel 同理)

#####Webpack 核心概念

  • entry 指定入口文件
    • 单入口
    • 多入口
  • output 指定输出相关信息
    • path 指定 处理过后 的文件 放到那个路径下
    • filename
      • 单入口 ‘index.js’
      • 多入口 ‘[name].js’
  • loader 帮助 webpack 处理 非JavaScript 文件
  • plugins 插件 相当于 一个工具 执行范围更广的任务

GET / POST 对比
  • GET 指 获取数据 POST指创建数据
  • GET在请求头 （网址）携带数据 POST 可以 网址（很少） 也可以通过请求体携带
  • GET会被 缓存 POST不会
  • 相对来说 POST稍微安全一些
HTTP状态码
  • 100 - 199 请求接受 继续处理
  • 200 - 299 成功
  • 300 - 399 重定向
  • 400 - 499 请求错误
  • 500 - 599 服务器错误

==Cookie 和 localStorage==

1. Cookie

  • 存储在用户本地， 不存储在服务器上

  • 可以随着浏览器每次请求 发送到 服务器上

######1.1 Cookie 用法

  • 操作
    • 写入
    • 读取
  • 属性
    • 名称(name) 值(value)
    • 失效时间
      • expires
      • max-age ( == 0 / -1 时 将会删除Cookies)
    • domain / path / secure
    • name, domain, path 都相同 才能说明是同一个Cookie
    • name value 必须填写 其他可以默认
    • 如果包含非英文，需要用 encodeURLComponent() 编码 写入时要用 decodeURLComponent() 解码
