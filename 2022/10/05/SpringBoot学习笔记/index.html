是什么

SpringBoot提供了一种快速使用Spring的方式，基于约定优于配置的思想，可以让开发人员不必在配置与逻辑业务之间进行思维的切换，全身心的投入到逻辑业务的代码编写中，从而大大提高了开发的效率。

为什么用SpringBoot

Spring缺点

配置繁琐

搭建ssm项目，需要配置大量xml。 application.xml spring-mybatis.xml spring-mvc.xml,大量的bean。

依赖繁琐

pom.xml要写大量依赖。 pom.xml spring-core spring-bean spring-mvc spring-mybatis java-connector

版本冲突 spring-core 4.0 spring-mvc 5.0

SpringBoot特点

  1. 自动配置
  2. 不需要代码生成，也不需要 XML 配置
  3. 定义好了依赖版本，避免了依赖冲突
  4. 辅助功能
    1. tomcat
    2. 外部配置

注意：Spring Boot 并不是对 Spring 功能上的增强，而是提供了一种快速使用 Spring 的方式。

需求：访问 localhost:8080/hello/test 返回 hello springboot

解释

  3. 使用SpringBoot和Spring构建的项目，业务代码编写方式完全一样。

起步依赖原理

ctrl + 左键 点击对应的依赖查看内部到底是什么

  1. spring-boot-starter-parent 的 parent -> spring-boot-dependencies中定义好了各种可能会用到的依赖
  2. 各种版本号也是规定好的，避免了版本问题
  3. spring-boot-starter-web也是同样的道理
    1. 其中包含了spring-mvc
    2. tomcat
    3. json等需要的依赖

所以看起来 是只需要导入这两个包 也是为什么把它叫做起步依赖的原因

SpringBoot快速构建

  1. new project的时候不选maven 而是选择 spring initializr
  2. next 后 依赖选择 选web -> springWeb
  3. 直接创建 等待依赖下载完成即可

配置文件

配置文件分类

springboot一个很重要的特点就是约定大于配置，默认配置文件就是

  1. application.properties
  2. application.yml
  3. application.yaml

三者的优先级 properties -> yml -> yaml

yml 使用最多 因为写法简洁，结构清晰 yaml同理 只是多了一个字母而已

yaml

是什么

YAML是一种直观的能够被电脑识别的的数据数据序列化格式，并且容易被人类阅读，容易和脚本语言交互的，可以被支持YAML库的不同的编程语言程序导入

特点

  1. 大小写敏感
  2. key: value key:后面必须有空格
  3. #是注释

profile运维

是什么

项目开发时候一般三种环境: 开发、测试、生产，profle就是用来动态切换三种环境的

基本使用

  1. 分别创建三种环境下的配置文件
    1. application-dev.yml
      1. 配置端口号 8080
    2. application-test.yml
      1. 配置端口号 8081
    3. application-pro.yml
      1. 配置端口号 8082
  2. 创建主配置文件
    1. application.yml

这时候激活的就是开发环境

注意控制台打印 The following 1 profile is active: “dev”

打包之后的激活方式

麻烦一点可以在IDEA改变 spring.profiles.active 来改变环境 然后重新打包

但一般做法是

  1. 使用maven 的 package 进行打包 jar包会在target目录下
  2. cmd输入 java -jar springboot-2-profile-0.0.1-SNAPSHOT.jar --spring.profiles.active=pro
  3. 需要不同环境直接在 = 号右边更改即可

其实就相当于在源代码中 改变主配置文件

配置文件的位置

项目内部配置文件

加载顺序为下文的排列顺序，高优先级配置的属性会生效

  • file:./config/：当前项目下的/config目录下
  • file:./ ：当前项目的根目录
  • classpath:/config/：classpath的/config目录
  • classpath:/ ：classpath的根目录

classpath == resources 只是打包之后 resources下的东西会放在 类路径下

注意

高级配置文件只覆盖低级配置文件的重复项。低级配置文件的独有项任然有效

项目外部配置文件

  1. 命令行

java -jar springboot-2-profile-0.0.1-SNAPSHOT.jar --spring.profiles.active=pro

  1. 指定配置文件位置

java -jar myproject.jar --spring.config.location=d://application.properties

简单说这种一般是打包成jar包后 想要在命令行改变配置文件 而不想重新打包

作用：生产环境，随时改变环境变量(dev/test/prod)时，可以通过改变配置文件来做。不需重新打包项目。
